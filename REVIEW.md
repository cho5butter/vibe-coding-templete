# PR レビュー指示 — Devin Review

このファイルは Devin の PR レビュー時に自動で参照される。

## レビュー観点

### 必須チェック

1. **TDD 遵守**: テストが実装より先に書かれているか（コミット履歴で確認）
2. **品質ゲート通過**: CI（`quality-gate.yml`）がパスしているか
3. **spec 整合性**: `spec/requirements.md` と `spec/design.md` の内容に沿った実装か
4. **spec 変更の合意**: `spec/` 内のファイルが変更されている場合、ユーザー合意が明示されているか

### コード品質

- セキュリティ脆弱性がないか（OWASP Top 10: インジェクション、XSS 等）
- 不要なファイル・デッドコードが含まれていないか
- エラーハンドリングが適切か

### コミットメッセージ

- 日本語で記述されているか
- 種別プレフィックス（`機能` / `修正` / `改善` / `整理` / `テスト` / `文書` / `設定` / `計画`）が付いているか
- MD ファイルがコミットから除外されていないか

## レビューコメントのスタイル

- 指摘は具体的に、修正案を含めて記述する
- 重要度を明示する: `[必須]` / `[推奨]` / `[提案]`
